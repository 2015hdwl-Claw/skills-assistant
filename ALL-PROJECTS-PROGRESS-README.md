# 📊 所有專案進度總覽 - 使用說明

## 🎯 功能概述

**所有專案進度總覽** 是一個強大的專案管理儀表板，能夠：
- 🔍 自動掃描 `C:\Users\ntpud\.claude\projects\` 下的所有專案
- 📈 統一顯示所有專案的進度、狀態、描述
- 🔄 支援標準化專案和個別專案的混合顯示
- 📊 提供專案間的進度比較和統計分析

## 🚀 快速開始

### 1. 掃描專案（首次使用或新增專案時）

```bash
# 在 skills-assistant 目錄下執行
node scan-all-projects.js

# 或使用快捷腳本
update-projects.bat
```

### 2. 查看進度總覽

```
http://localhost:8080/all-projects-progress.html
```

## 🎮 功能特色

### 📊 統計概覽
- 總專案數：3 個標準格式專案
- 平均進度：即時計算
- 進行中專案：即時統計
- 已完成專案：即時統計

### 🔍 搜尋與篩選
- **即時搜尋**：按專案名稱、描述搜尋
- **狀態篩選**：全部、進行中、已完成、規劃中
- **進度排序**：比較視圖中按進度排序

### 📈 視覺化功能
- **專案卡片**：顯示進度條、階段、狀態
- **比較視圖**：水平進度條比較圖
- **統計圖表**：數字動畫更新效果

### 🖱️ 可點擊專案卡片
- **一鍵查看詳情**：點擊任何專案卡片即可查看完整專案詳情
- **視覺提示**：滑鼠懸停顯示"👆 點擊查看詳情"
- **自動導航**：直接導航到對應專案的詳情頁面
- **完整資訊**：專案詳情頁面包含完整的進度、階段、技能和時間線資訊

### 💾 資料管理
- **資料匯出**：匯出 JSON 格式的進度報告
- **重新載入**：更新顯示資料
- **重新掃描**：執行完整專案掃描

## 🔄 更新流程

### 自動更新專案索引
```bash
node scan-all-projects.js
```

### 更新瀏覽器顯示
- 點擊頁面上的 **🔄 重新載入** 按鈕
- 或重新整理瀏覽器頁面 (F5)

## 📁 專案類型

### 標準化專案
位於 `project-plans/` 目錄，包含：
- `{PROJECT_NAME}_PROGRESS.json` - 進度追蹤
- `{PROJECT_NAME}_YC_PLAN.md` - YC 5-phase 計畫書
- `README.md` - 專案說明
- `CLAUDE.md` - 專案規則
- `{PROJECT_NAME}_VERSIONS.json` - 版本歷史

### 個別專案
位於 `projects/` 目錄下的各個專案資料夾，包含：
- README.md
- CLAUDE.md
- 任何進度或計畫檔案

## 🖱️ 專案詳情查看流程

### 從總覽到詳情的完整工作流程

1. **查看所有專案**
   ```
   http://localhost:8080/all-projects-progress.html
   ```
   - 看到所有標準格式專案的卡片
   - 每個卡片顯示：專案名稱、進度、狀態、當前階段

2. **點擊專案卡片**
   - 滑鼠懸停在專案卡片上會顯示 "👆 點擊查看詳情"
   - 卡片會有視覺回饋（陰影加強、輕微上移）
   - 點擊後自動導航到專案詳情頁面

3. **查看專案詳情**
   ```
   自動導航到: project-detail.html?project={專案名稱}
   ```
   - 完整的專案概覽
   - 詳細的進度資訊
   - YC 5-Phase 階段分析
   - 推薦技能和下一步行動
   - 專案時間線

4. **返回總覽頁面**
   - 點擊詳情頁面左上角的 "← 返回總覽" 按鈕
   - 或使用瀏覽器的後退功能

### 🔗 導航鏈接格式

每個專案卡片對應的詳情頁面鏈接：
```
project-detail.html?project=admission-master      # 升學大師
project-detail.html?project=social               # 社群經營大師
project-detail.html?project=project-planning-master # 專案規劃大師
```

## 🎨 介面說明

### 專案卡片內容
- **專案標題**：顯示名稱和 ID
- **專案描述**：從 README 或計畫書提取
- **進度條**：整體完成百分比
- **階段時間軸**：YC 5-phase 階段進度
- **專案元資料**：當前階段、專案類型
- **資料來源**：顯示可用的專案檔案

### 狀態徽章
- **🟢 已完成**：進度 100%
- **🟡 進行中**：進度 1-99%
- **🟣 規劃中**：有計畫但無進度
- **⚪ 未知**：無法判斷狀態

## ⚡ 快捷鍵

- **Ctrl + F**：聚焦搜尋框
- **Ctrl + R**：重新載入資料

## 🔧 疑難排解

### 看不到專案
1. 確認已執行 `node scan-all-projects.js`
2. 檢查 `all-projects-index.json` 是否存在
3. 確認 HTTP 伺服器正在運行

### 顯示舊資料
1. 執行 `node scan-all-projects.js` 重新掃描
2. 點擊頁面上的 **🔄 重新載入** 按鈕

### 專案資料不完整
- 確保專案目錄中有對應的檔案（README.md、PROGRESS.json 等）
- 檢查檔案格式是否正確

## 📞 技術支援

如有問題，請檢查：
1. Node.js 是否正確安裝
2. 專案路徑是否正確
3. 檔案權限是否正常
