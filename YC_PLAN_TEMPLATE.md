### 標準開發循環

3. Code Review (code-reviewer agent)
4. 整合測試
5. 部署驗證

### 開發原則
- **持續 Code Review**: 每個功能完成後立即審查
- **TDD 優先**: 先寫測試，再寫代碼
- **小型頻繁**: 小功能單元，頻繁審查，快速迭代
- **質量優先**: 通過 code review 才能合併
